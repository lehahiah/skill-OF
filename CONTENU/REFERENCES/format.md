## Diaporama

### Structure type (15-20 slides)

```
[SLIDE 1] TITRE
• Titre accrocheur (pas "Formation sur X" mais "Pourquoi X change tout pour Y")
• Sous-titre : contexte ou promesse
• 💡 Visuel : image forte, chiffre clé, ou question rhétorique

[SLIDE 2] ACCROCHE / CONSTAT
• 1 chiffre ou fait qui crée la prise de conscience
• 1 phrase de contexte
• 💡 Visuel : stat mise en forme graphiquement

[SLIDE 3] PLAN / SOMMAIRE
• 3 à 5 parties max, formulées comme des promesses
• Ex : "1. Comprendre pourquoi ça échoue — 2. Les 3 leviers qui changent tout — 3. Votre plan en 30 jours"

[SLIDES 4 à N-3] CORPS DU CONTENU
Structure par partie :
• 1 slide de titre de partie (impact visuel)
• 2 à 4 slides de contenu avec : points courts / schéma / exemple / cas pratique
• 1 slide "Ce qu'on croit / Ce qui est vrai" ou "Erreur fréquente"

[SLIDE N-2] SYNTHÈSE
• 3 à 5 points clés à retenir
• Formulés comme des actions ou des principes, pas des résumés passifs

[SLIDE N-1] PASSAGE À L'ACTION
• Ce que le participant peut faire dès demain
• Outil, ressource, ou étape concrète

[SLIDE N] CONTACT / RESSOURCES
• Coordonnées, lien, QR code
• Question ouverte pour prolonger
```

### Règles de mise en forme texte pour slides

- Titre de slide : ≤ 8 mots, verbe d'action ou formulation-choc
- Contenu : 3-4 points max, ≤ 12 mots par point
- Pas de phrases complètes dans les bullets — fragments actifs
- 1 idée = 1 slide (si trop long → couper en 2 slides)
- Utiliser la règle "Est-ce qu'on peut lire ça en 5 secondes ?" — si non, couper

## Article

### Structure type

```
[CHAPEAU]
2-3 lignes. Accroche qui crée une tension, une question, ou un constat inattendu.
Exemple : "90 % des plans de formation sont validés sans jamais être évalués. Voici pourquoi — et comment faire autrement."

[INTERTITRE 1] — Formulé comme une question ou une promesse
Corps : 150-250 mots, 1 idée principale, 1 exemple ou illustration
Optionnel : 1 encadré (définition, zoom, chiffre clé)

[INTERTITRE 2]
Corps : idem
[SOURCE] si données citées

[INTERTITRE 3]
Corps : cas pratique ou application concrète
Peut inclure une mini check-list

[INTERTITRE 4 — optionnel]
Erreurs fréquentes ou points de vigilance

[SYNTHÈSE]
3 phrases max. Ce qu'on retient. Ce qu'on peut faire.
Pas de "En conclusion, nous avons vu que..." — trop scolaire.
```

### Règles éditoriales article

- Phrases courtes : 1 idée par phrase, < 25 mots en moyenne
- Alterner : affirmation forte → nuance → exemple
- Éviter les tournures passives et le conditionnel de prudence excessive
- Les intertitres doivent donner envie de lire la section, pas la résumer
- Longueur idéale : 700-1200 mots (article blog/LinkedIn) ou 1500-2500 (article expert)

## Carrousel

### Structure type (8-12 slides)

```
[SLIDE 1] ACCROCHE
• 1 seule phrase, 5-10 mots max
• Doit provoquer : question dérangeante, chiffre surprenant, affirmation contre-intuitive
• Exemple : "Vos formations ne servent à rien. Voici pourquoi."
• 💡 Visuel : fond coloré + typographie grande

[SLIDE 2] PROMESSE / PLAN
• "Dans ce carrousel, tu vas découvrir…"
• 3-4 points, très courts

[SLIDES 3 à N-1] CORPS
Chaque slide = 1 idée forte
Formats efficaces par slide :
  - ❌ Mythe / ✅ Réalité
  - Avant / Après
  - 1 chiffre + 1 phrase d'explication
  - "La plupart font X. Les meilleurs font Y."
  - Étape N : [titre action] + 1-2 lignes

[SLIDE N] CTA
• Question ouverte pour l'engagement : "Et toi, tu fais comment ?"
• OU : "Sauvegarde ce carrousel pour y revenir"
• OU : lien vers ressource / offre
```

### Règles de ton carrousel

- Direct, sans condescendance
- "Tu" (pas "vous") sauf si audience clairement formelle
- Formulations nettes : pas de "peut-être", "il semblerait", "dans certains cas"
- Opposition = clarté (le lecteur comprend en 2 secondes)
- Chaque slide doit être autonome (compréhensible sans les autres)

## Fiche mémo

### Structure type (format 1 page A4 ou A5)

```
[EN-TÊTE]
Titre : [Sujet] — Le guide en un coup d'œil
Sous-titre : [Public + contexte]
Date / Version (si utile)

[BLOC 1] DÉFINITION / CONTEXTE
2-3 lignes max. Ce qu'il faut savoir pour tout comprendre.

[BLOC 2] LES ÉTAPES / LA MÉTHODE
Format : liste numérotée, 4-7 étapes
Chaque étape : Titre court (3-5 mots) + 1 ligne d'action concrète

[BLOC 3] POINTS CLÉS À RETENIR
Format : puces courtes, ≤ 10 mots par puce
5-7 points max

[BLOC 4] ERREURS FRÉQUENTES
Format : ❌ [Erreur] → ✅ [Bonne pratique]
3-5 erreurs

[BLOC 5] RESSOURCES / RÉFÉRENCES
Lois, textes, outils, liens — avec courte description

[PIED DE PAGE]
Logo / auteur / contact
```

### Règles fiche mémo

- Densité maximale, zéro remplissage
- Chaque mot compte — si une phrase peut être coupée en 2 mots, le faire
- Hiérarchie visuelle claire : titres de bloc ≠ contenu
- Peut être imprimée sans perte d'information
- Test : "En 90 secondes, je comprends l'essentiel ?" — si non, recouper

## Note client

### Structure type

```
[EN-TÊTE]
À l'attention de : [Nom / Fonction / Structure]
De : [Auteur]
Objet : [Titre explicite, pas générique]
Date :

[CONTEXTE / RAPPEL]
1-2 paragraphes : pourquoi cette note, sur quoi elle porte, dans quel cadre.

[DIAGNOSTIC / ÉTAT DES LIEUX]
Ce qui a été observé / analysé / constaté.
Structuré par thématique ou par ordre d'importance.
Distinguer : fait établi / interprétation / recommandation

[PRÉCONISATIONS]
Hiérarchisées : priorité 1 → priorité 2 → priorité 3
Pour chaque préconisation :
- Ce qui est recommandé
- Pourquoi (logique ou risque évité)
- Comment (niveau d'effort / ressources / calendrier)

[PROCHAINES ÉTAPES]
Actions concrètes, responsables, délais.

[ANNEXES]
Si nécessaire : données, références, outils
```

### Règles note client

- Ton : professionnel mais lisible — pas de jargon inutile
- Chaque recommandation doit être argumentée, pas imposée
- Distinguer clairement ce qui est certain de ce qui est hypothèse
- Le client doit pouvoir lire la synthèse sans lire l'intégralité

## Programme

### Structure type (formation ou webinaire)

```
[EN-TÊTE]
Titre : [Accrocheur, orienté résultat]
Durée : [X heures / X modules / X sessions]
Public : [qui, prérequis]
Objectifs pédagogiques :
  - À l'issue de cette formation, le participant sera capable de…
  - (3-5 objectifs, formulés avec des verbes d'action : identifier, appliquer, construire, évaluer)

[MODULE 1] TITRE DU MODULE
  Durée : Xh
  Contenu :
    - Concept 1 (définition + exemple)
    - Concept 2
    - Mise en pratique / exercice
  Livrables / sorties : [ce que le participant produit ou sait faire]

[MODULE 2] …

[ÉVALUATION]
  Type : [QCM / mise en situation / projet / quizz]
  Critères : [ce qui est évalué]

[MODALITÉS]
  Présentiel / distanciel / mixte
  Supports fournis
  Prérequis techniques si distanciel
```

### Règles programme

- Les objectifs commencent toujours par un verbe d'action (Bloom)
- Chaque module = 1 compétence ou 1 capacité nouvelle
- Les exercices et mises en pratique sont décrits, pas juste mentionnés
- Le programme doit répondre à la question : "Qu'est-ce que je sais faire à la fin ?"
